Home page

Welcome to Day to Day Welcome to Day to Day, a daily planner meant to help with planning out what events, tasks, and projects you need to attend or work on each day. Day to Day provides an easy way to make sure that everything that needs to be done will get done. Its different task types allow for a broad range of planning strategies. With Day to Day, you can plan out your day the way you want to. Task Types Events Events are time-specific tasks, and is used to block out time in many planning strategies. The Event task type allows you to set a specific time for something, be it an actual event, a task, or a habit. Events can also be set to have a repeat. Deadlines Deadlines are tasks with a time-specific end, such as a project in work or homework in school. The Deadline task type allows you to set a deadline for the task, its expected workload, and the work strategy you would like to employ. Day to Day will then automatically add it to your to-do list, accounting for the workload and work strategy. Repeats Repeats are tasks that are to be occomplished every so often, according to the frequency that the task is set at. This is used for tasks that have to be regularly acomplished.

Task Types Events Events are time-specific tasks, and is used to block out time in many planning strategies. The Event task type allows you to set a specific time for something, be it an actual event, a task, or a habit. Events can also be set to have a repeat. Components: Date: The day of that the task is to be occomplished Start time: The time that the task starts. End time: The time that the task ends. Frequency: How often the task should be repeated, is set to none by default Deadlines Deadlines are tasks with a time-specific end, such as a project in work or homework in school. The Deadline task type allows you to set a deadline for the task, its expected workload, and the work strategy you would like to employ. Day to Day will then automatically add it to your to-do list, accounting for the workload and work strategy. Components: Date: The day that the task ends. Time: The time the task ends. Workload: The number of hours that the task is expected to take Work strategy: How the hours are spread. Backload: All hours are planned as close to the deadline as possible. Frontload: all hours are planned as soon as possible. Spread: All hours are spread as evenly as possible until the deadline. Even spread: All hours are spread out to even out the total number of hours working each day. Repeats Repeats are tasks that are to be occomplished every so often, according to the frequency that the task is set at. This is used for tasks that have to be regularly acomplished, and is displayed in the to-do list. Components: Frequency: How often the task will be on the to-do list.
